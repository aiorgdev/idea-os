# Validation Criteria Reference

Scoring guidelines for Idea OS validation assessments.

## Problem Validation Criteria

### Real (Is the problem real?)

| Score | Criteria |
|-------|----------|
| 10 | 100% of interviewees confirm; abundant evidence online |
| 9 | 90%+ confirm; strong external evidence |
| 8 | 80%+ confirm; good external evidence |
| 7 | 70%+ confirm; some external evidence |
| 6 | 60%+ confirm; limited external evidence |
| 5 | 50% confirm; mixed signals |
| 4 | 40% confirm; more don't have problem |
| 3 | 30% confirm; minority issue |
| 2 | Few confirm; edge case |
| 1 | No one confirms; not a real problem |

## Commitment Signals

### Strong (Score boost)

- Offering to pay/pre-order (+2)
- Asking for specific availability (+1)
- Making introductions to buyers (+1)
- Sharing internal data/documents (+1)
- Spending significant time (30+ min) (+1)

### Weak (No adjustment)

- "Sounds interesting"
- "Would probably use it"
- Asking general questions
- Polite enthusiasm

### Negative (Score reduction)

- Multiple objections (-1)
- Comparing unfavorably to alternatives (-1)
- No follow-up questions (-1)
- Short interview, quick exit (-2)
